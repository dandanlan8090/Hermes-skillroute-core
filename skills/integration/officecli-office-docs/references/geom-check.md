# PPTX 坐标级几何校验（vision 漏报兜底）

## 为什么需要
本会话（2026-07-13）做 16 页讲解 PPT 时，user 连续反馈以下视觉问题：
- 第 2 页：两个圆角框重叠、下面字体与框边缘重叠
- 第 4 页：底部三个圆角框超出 PPT
- 第 5 页：黄色柱形与文字重叠
- 第 11 页：没对齐、超出页面边界
- 第 14 页：两个圆角框重叠

而 `vision_analyze` 对这每一页都曾判「无问题 / 间距舒适」。
**结论：vision 对轻微交叠、贴边、超出画布这类几何异常会漏报。** 凡是 user 说"重叠/超出/贴边"，必须做坐标级硬校验，不能只信 vision。

## 画布上限（16:9 EMU）
- 宽度上限：12192000 EMU
- 高度上限：6858000 EMU
- 单形状判定超界：`x + cx > 12192000` 或 `y + cy > 6858000`
- 多卡片页建议：单卡 `cx ≤ 11000000`、`y + cy` 距底边留 ≥300000 余量

## 重叠判定
- 两矩形 A(x1,y1,cx1,cy1) 与 B(x2,y2,cx2,cy2)：
  - 交叠宽 `ix = max(0, min(x1+cx1, x2+cx2) - max(x1, x2))`
  - 交叠高 `iy = max(0, min(y1+cy1, y2+cy2) - max(y1, y2))`
  - 判重叠：`ix > 容差 and iy > 容差`
- **容差建议 50000 EMU**（约 0.04cm）。圆角矩形（roundRect adj≈10000-60000）视觉留白会让实际矩形边往内收，50k 容差可避免把视觉留白误报为重叠。
- **有意叠放要跳过**：表格行底色 `<p:sp>`（如 Row210）与右侧数值文字 `<p:sp>`（如 Val210）坐标故意相同，属设计，检测时按 name 前缀（Row*/Val*、Deco*、Part*、TB*、SB*）过滤掉，不当 bug 修。

## 检测脚本
`scripts/geom_check.py <pptx路径>` —— 输出每页的超界形状清单 + 两两重叠清单（已按 name 前缀跳过装饰/标题/有意叠放）。可直接复跑。

## 本会话真实踩坑（已验证的坐标修复）
| 页 | 现象 | 根因（坐标） | 修复 |
|----|------|----------|------|
| 2 | 两栏重叠+贴边 | 两栏 x 间距仅 350000，说明 y 距卡底 -100000 | 两栏各缩宽、间距拉到 200000；说明下移 250000 |
| 4 | 底部三卡超界 | Card213-215 `y+cy=7000000 > 6858000` | 6 卡 cy 2400000→1900000，行距拉开，底 6050000 安全 |
| 5 | 黄条与文字重叠 | Bar211(y3200000) 与 SB220(y3250000) 同 y 区间且 x 重叠 | 条形缩高上移、说明移到条形下方留白、五类小卡下移 |
| 11 | 气泡超出右界 | Agent 气泡 `x=7200000 cx=6200000` → 右 13400000 > 12192000 | 改 `x=7000000 cx=6200000`（与用户气泡左对齐） |
| 14 | 两栏重叠 | 间距 350000 | 缩宽至 4000000、间距 200000 |
| 插入页 | slide 索引冲突 | 中途插 Slide12 后其后 add_slide /slide[12] 重复 | 其后全部 /slide[N] +1 顺延，echo 文案同步改 |

## 经验
1. user 反馈的视觉问题永远优先于 vision 的「通过」结论。
2. 先坐标硬校验定位，再改 raw-set 坐标/尺寸，最后才用 vision 做最终确认。
3. 多卡片页务必先用坐标表预先排布，避免手改时卡片互相压住。
4. 插入页必须整体顺延其后所有 slide 索引，否则生成文件会丢页或冲突。
